# Theory notes

## Fixed-prime smooth numbers as a log-lattice

For a fixed prime set `P = {p1, ..., pk}`, every `P`-smooth number has a unique exponent vector:

```text
x = p1^e1 * ... * pk^ek,  ei >= 0.
```

The increasing order of the integers is the increasing order of the log weights:

```text
log x = e1 log p1 + ... + ek log pk.
```

The rank/count problem is therefore a lattice-point count under a hyperplane:

```text
C_P(T) = #{ e in N^k : sum_i e_i log p_i <= T }.
```

## Exponent-stream self-similarity

Let `S_P` be the ordered list of all `P`-smooth numbers. For a fixed prime `pi`, write `E_i(n)` for the exponent of `pi` in the `n`-th entry.

Deleting the zero entries of `E_i` and subtracting 1 from every remaining entry produces the same exponent stream again. Reason: every entry with positive `pi`-exponent is `pi * y` for another `P`-smooth number `y`, and division by `pi` preserves order.

## Zero positions as lower-dimensional faces

The zeroes in the `pi`-exponent stream are exactly the entries not divisible by `pi`; equivalently, the smooth numbers generated by `P \ {pi}`. Their positions in the full stream are given by the rank of those lower-dimensional face points inside the full `P` ordering.

## Algorithmic consequences

- `k = 3`: count by summing fast two-prime floor counts over the third exponent.
- `k = 5`: represent one 3-prime MITM side as shifted layers of a 2-prime base.
- higher `k`: use MITM-style sumset selection and delay exponent reconstruction.

The implementation is intentionally conservative: fast kernels are benchmarked, while headline examples can be independently audited using interval-rank counting.

## Analytic bracketing as a non-MITM hybrid

The counting function has a useful asymptotic expansion obtained from the
Laplace/generating-function identity

```text
sum_e exp(-s * <e, log P>) = product_i (1 - exp(-s log p_i))^-1.
```

Expanding

```text
(s w) / (1 - exp(-s w)) = 1 + s w / 2 + (s w)^2 / 12 - (s w)^4 / 720 + ...
```

gives a polynomial approximation to

```text
C_P(T) = #{e in N^k : <e, log P> <= T}.
```

The layer-compressed solver uses this approximation only to choose an initial
large-rank search bracket. It does not use the approximation as evidence of
correctness: the bracket is still checked and repaired by exact layer counts,
and the returned exponent vector still needs the independent interval auditor
for certification.

The analytic-count residual workbench evaluates the same expansion at certified
target exponent vectors and compares it with the certified rank. This is a
diagnostic step toward an analytic count oracle; it is not yet a theorem or a
replacement for exact counting.
